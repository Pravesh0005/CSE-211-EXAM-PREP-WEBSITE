## ⚙️ Unit Breakdown

<details>
<summary><b>⚙️ Unit 1 — Basic Computer Organization</b></summary>

### Core Concepts
- **Registers (9 types)**: PC (12-bit), AR (12-bit), IR (16-bit), DR (16-bit), AC (16-bit), TR (16-bit), INPR (8-bit), OUTR (8-bit), SC (4-bit)
- **Instruction Format (16-bit)**: Bit15=I (addressing mode), Bits14-12=Opcode (3-bit→8 opcodes), Bits11-0=Address (12-bit)
- **Memory Reference Instructions**: AND, ADD, LDA, STA, BUN, BSA, ISZ
- **Register Reference Instructions** (opcode=111, I=0): CLA, CLE, CMA, CME, CIR, CIL, INC, SPA, SNA, SZA, SZE, HLT
- **I/O Instructions** (opcode=111, I=1): INP, OUT, SKI, SKO, ION, IOF

### Fetch Cycle (Every Instruction)
```
T0: AR ← PC
T1: IR ← M[AR], PC ← PC+1
T2: Decode IR(12-14), AR ← IR(0-11), I ← IR(15)
```

### Common Bus (S2S1S0 encoding)
```
001=AR  010=PC  011=DR  100=AC  101=IR  110=TR  111=Memory
```

</details>

<details>
<summary><b>⚙️ Unit 2 — Central Processing Unit</b></summary>

### Addressing Modes (EA Formulas)
| Mode | EA Formula | Example |
|------|-----------|---------|
| Direct | `EA = address field` | `LDA 500` |
| Indirect | `EA = M[address field]` | Extra memory access |
| PC Relative | `EA = PC + offset` | PC=400, offset=20 → EA=420 |
| Indexed | `EA = IX + offset` | IX=1000, offset=50 → EA=1050 |
| Immediate | Operand IS the field | `MVI B, 44h` |

### RISC vs CISC
| Feature | RISC | CISC |
|---------|------|------|
| Control | Hardwired ⚡ | Microprogrammed |
| Instructions | Fixed-length | Variable-length |
| Memory | Load-Store only | Memory-to-memory |
| Pipeline | Excellent | Difficult |
| Examples | ARM, MIPS | Intel x86 |

### Stack Operations
```
PUSH: SP ← SP-1, M[SP] ← DR    (decrement FIRST)
POP:  DR ← M[SP], SP ← SP+1   (read FIRST)
Stack grows DOWNWARD. SP → TOP of stack.
```

</details>

<details>
<summary><b>⚙️ Unit 3 — I/O Organization</b></summary>

### I/O Transfer Modes
| Mode | CPU Role | Efficiency |
|------|---------|------------|
| Programmed I/O | Busy-wait polling | ❌ Very low |
| Interrupt-Driven | Works, interrupted when ready | ⚡ Medium |
| **DMA** | Only initializes; free during transfer | ✅ High |
| IOP/Channel | Initiates; IOP handles everything | ✅ Highest |

### DMA Registers
- **Address Register** — starting memory address
- **Word Count Register** — number of words to transfer
- **Control Register** — read/write direction

### Daisy Chain Priority
```
Device with PI=1, NO pending  → PO=1 (passes priority through)
Device with PI=1, HAS pending → PO=0 (stops chain, generates interrupt)
Closest to CPU = HIGHEST priority
```

</details>

<details>
<summary><b>⚙️ Unit 4 — RTL & Computer Arithmetic</b></summary>

### Logic Micro-operations
| Operation | RTL | Effect |
|-----------|-----|--------|
| Selective Set | `A ← A ∨ B` | Set bits where B=1 |
| Selective Complement | `A ← A ⊕ B` | Flip bits where B=1 |
| Selective Clear | `A ← A ∧ B'` | Clear bits where B=1 |
| Mask | `A ← A ∧ B` | Clear bits where B=0 |

### Booth's Algorithm
```
Qn=1, Q(n-1)=0  → SUBTRACT multiplicand, then ASHR
Qn=0, Q(n-1)=1  → ADD multiplicand, then ASHR
Qn=Qn-1 (00/11) → Just ASHR
Start: Q(n-1) = 0, AC = 0
```

</details>

<details>
<summary><b>⚙️ Unit 5 — Memory Organization</b></summary>

### Cache Hit Time Formula
```
Te = Tc + (1-h) × Tm
```
> **Example**: Tc=0.4ns, Tm=1.2ns, h=0.85 → Te = 0.4 + (0.15)(1.2) = **0.58ns**

### Cache Mapping
| Type | Address | Hardware | Conflict? |
|------|---------|----------|-----------|
| Direct | Tag + Index + Offset | Simple RAM | High ⚠️ |
| Associative | Tag + Offset | CAM (parallel) | None ✅ |
| Set-Assoc | Tag + Set + Offset | Medium | Low ✅ |

### Page Replacement Algorithms
| Algorithm | Rule | Belady's Anomaly? |
|-----------|------|-------------------|
| FIFO | Oldest page out | ⚠️ YES |
| LRU | Least recently used | ❌ No |
| OPT | Not used longest future time | ❌ No (optimal, impractical) |

</details>

<details>
<summary><b>⚙️ Unit 6 — Parallel & Modern Architecture</b></summary>

### Pipeline Speedup
```
Speedup = (n × k) / (k + n - 1)
As n → ∞: Speedup → k (number of stages)

5-stage RISC: IF → ID → EX → MEM → WB
```

### Flynn's Taxonomy
| Class | Instructions | Data | Example |
|-------|-------------|------|---------|
| SISD | Single | Single | Traditional CPU |
| SIMD | Single | Multiple | GPU, Vector |
| MISD | Multiple | Single | Rare/Theoretical |
| MIMD | Multiple | Multiple | Multiprocessors |

</details>

## ⚠️ Critical Traps

> These are the **most commonly wrong answers** in COA exams

```diff
- ❌ WRONG: PC holds current instruction
+ ✅ RIGHT: PC holds address of NEXT instruction (IR holds current)

- ❌ WRONG: Stack grows upward (SP increments on PUSH)
+ ✅ RIGHT: Stack grows DOWNWARD (SP decrements on PUSH, increments on POP)

- ❌ WRONG: RISC uses microprogrammed control
+ ✅ RIGHT: RISC uses HARDWIRED control. CISC uses microprogrammed.

- ❌ WRONG: Cache Te = h×Tc + (1-h)×Tm
+ ✅ RIGHT: Cache Te = Tc + (1-h)×Tm  (Tc always accessed)

- ❌ WRONG: DMA initializes itself
+ ✅ RIGHT: CPU initializes DMA (address + word count + control), then CPU is free

- ❌ WRONG: Daisy Chain PI=1, PO=0 means no interrupt
+ ✅ RIGHT: PI=1, PO=0 means device IS requesting interrupt (blocks chain)

- ❌ WRONG: Direct mapping has no conflict misses
+ ✅ RIGHT: Direct mapping suffers MOST from conflict misses

- ❌ WRONG: FIFO is the best page replacement
+ ✅ RIGHT: LRU is best practical. OPT is theoretical best. FIFO has Belady's anomaly.

- ❌ WRONG: Booth's starts with Q(n-1)=1
+ ✅ RIGHT: Booth's ALWAYS starts with Q(n-1)=0

- ❌ WRONG: IOP = DMA with more memory
+ ✅ RIGHT: IOP can FETCH and EXECUTE its own I/O instructions (DMA cannot)
```

## 📐 Key Formulas at a Glance

```
Cache Effective Time:    Te = Tc + (1-h) × Tm
Pipeline Speedup:        S  = (n × k) / (k + n - 1)
Total Factors of N:      (p+1)(q+1)(r+1) for N=A^p × B^q × C^r
Stack PUSH:              SP ← SP-1, M[SP] ← DR
Stack POP:               DR ← M[SP], SP ← SP+1
EA (PC Relative):        EA = PC + offset
EA (Indexed):            EA = Index Register + offset
EA (Indirect):           EA = M[address field]
Interrupt condition:     IEN = 1 AND (FGI=1 OR FGO=1)
Fetch T0:                AR ← PC
Fetch T1:                IR ← M[AR], PC ← PC+1
DMA cycle-steal:         1 word at a time from CPU bus
Daisy priority:          Closest device to CPU = highest
RISC pipeline:           IF → ID → EX → MEM → WB
```
