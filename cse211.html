⚠️ Critical Traps & Confusions
TRAP 1: IR vs PC vs AR vs DRPC = address of NEXT instruction. AR = current address being accessed. IR = holds current instruction. DR = data read from/written to memory. AC = accumulator (computation). TR = temporary.
TRAP 2: Fetch Cycle TimingT0: AR←PC. T1: IR←M[AR], PC←PC+1. T2: Decode. These 3 timing signals are for FETCH. Don't mix with execute cycle timings.
TRAP 3: Register Reference vs Memory ReferenceRegister Reference: opcode=111, I=0 (D7=1, I'=1). I/O Instructions: opcode=111, I=1 (D7=1, I=1). Memory Reference: opcode≠111.
TRAP 4: Direct vs Indirect Addressing EADirect: EA = address field of instruction. Indirect: EA = M[address field]. Register Indirect: EA = [R]. PC Relative: EA = PC + offset.
TRAP 5: DMA vs Interrupt I/O vs Programmed I/OProgrammed I/O: CPU polls (busy-wait). Interrupt I/O: Device interrupts CPU when ready. DMA: Data transferred directly memory↔device WITHOUT CPU. CPU most efficient use = DMA.
TRAP 6: Cache Hit Ratio FormulaTe = h×Tc + (1-h)×(Tc+Tm) = Tc + (1-h)×Tm. NOT Te = h×Tc + (1-h)×Tm. Always add Tc for cache access regardless of hit/miss.
TRAP 7: RISC vs CISC Key PointsRISC: hardwired control (NOT microprogrammed), fixed-length instructions, load-store only. CISC: microprogrammed control, variable-length, memory-to-memory operations.
TRAP 8: Booth's Algorithm RulesCurrent bit=0, Prev bit=1 → ADD multiplicand. Current=1, Prev=0 → SUBTRACT. Current=Previous (00 or 11) → just arithmetic RIGHT SHIFT. Start with prev=0.
TRAP 9: Stack PUSH vs POPStack grows downward. PUSH: SP←SP-1, M[SP]←DR. POP: DR←M[SP], SP←SP+1. SP always points to TOP of stack (last pushed item).
TRAP 10: Daisy Chain PriorityDevice with PI=1 and no pending interrupt → passes PO=1 to next device. Device with PI=1 AND pending interrupt → keeps PO=0 (blocks chain). Highest priority = closest to CPU.

📨 Instruction Format (16-bit) MCQ

Bit 15 (I): Addressing mode (0=Direct, 1=Indirect) Bits 14-12 (Opcode): 3 bits → 8 possible opcodes (000 to 111) Bits 11-0 (Address): 12 bits → 4096 (2¹²) memory locations Memory Reference Instructions (opcode 000-110, I=0 or 1): AND, ADD, LDA, STA, BUN, BSA, ISZ Register Reference Instructions (opcode=111, I=0): CLA, CLE, CMA, CME, CIR, CIL, INC, SPA, SNA, SZA, SZE, HLT I/O Instructions (opcode=111, I=1): INP, OUT, SKI, SKO, ION, IOF

🔄 Instruction Cycle MOST TESTED

Fetch Cycle (every instruction):

T0: AR ← PC (Load PC into Address Register) T1: IR ← M[AR], PC ← PC+1 (Fetch instruction, increment PC) T2: Decode IR(12-14), AR ← IR(0-11), I ← IR(15) (Decode opcode + address)

If Indirect Addressing (I=1, Memory Reference, T3):

D'7IT3: AR ← M[AR] (Get effective address from memory)

Interrupt Cycle (if IEN=1 and FGI/FGO=1):

AR ← 0, TR ← PC (Save PC to temp) M[AR] ← TR, PC ← 0 (Store return address at memory[0], jump to ISR) PC ← PC+1, IEN ← 0 (Disable further interrupts, R ← 0)
TRAP: Fetch = T0,T1,T2 (3 signals). Total for memory-reference indirect = T0–T5 (6 signals). T0: AR←PC always first.

🚌 Common Bus System MCQ

7 registers + memory connected via 16-bit bus. Multiplexers select which register drives the bus using 3 control lines S2, S1, S0.

S2 S1 S0 → Register (Output to Bus) 0 0 0 → None (no selection) 0 0 1 → AR (Address Register) 0 1 0 → PC (Program Counter) 0 1 1 → DR (Data Register) 1 0 0 → AC (Accumulator) 1 0 1 → IR (Instruction Register) 1 1 0 → TR (Temp Register) 1 1 1 → Memory (M[AR])

🔌 I/O and Interrupt Flags MCQ

  • FGI (Input Flag): =1 means new input data available. =0 means computer accepted it.
  • FGO (Output Flag): =1 means data transferred from AC to OUTR. =0 means don't load new data.
  • IEN (Interrupt Enable): =1 enables interrupts, =0 disables interrupts.
  • Interrupt initiated when: IEN=1 AND (FGI=1 OR FGO=1)
TRAP: Interrupt flip-flop R is SET when interrupts occur and CLEARED during interrupt cycle. IEN is cleared during interrupt cycle to prevent nested interrupts.
2️⃣ Unit 2 — Central Processing Unit

🗂 General Register Organization CONTROL WORD

Modern CPUs have multiple general-purpose registers (R1–R7). Control unit directs info flow through ALU using:

Control Word = SELA (3 bits) | SELB (3 bits) | SELD (3 bits) | OPR (5 bits) SELA → selects source for Bus A (input to ALU) SELB → selects source for Bus B (input to ALU) SELD → selects destination register (from ALU output) OPR → ALU operation (ADD, SUB, AND, OR, XOR, TSFA, SHLA, etc.) Example: R1 ← R2 + R3 Control Word: SELA=R2, SELB=R3, SELD=R1, OPR=ADD = 010 | 011 | 001 | 00010

📚 Stack Organization PUSH/POP MCQ

Stack = LIFO structure. SP (Stack Pointer) points to TOP of stack. Stack grows DOWNWARD (SP decrements on PUSH).

PUSH (store DR onto stack): SP ← SP - 1 (decrement first) M[SP] ← DR (then store) if (SP=0) FULL ← 1 (check overflow) POP (retrieve from stack into DR): DR ← M[SP] (read first) SP ← SP + 1 (then increment) if (SP=0) EMPTY ← 1 (check underflow) Stack Overflow: PUSH when FULL=1 Stack Underflow: POP when EMPTY=1

Reverse Polish Notation (RPN/Postfix): A B + C × evaluates left-to-right using stack. Operands pushed, operators pop and compute.

Example: (3×4) + (5×6) in RPN = 3 4 × 5 6 × +

🚨 Interrupts & PSW TYPES MCQ

Types of Interrupts:

  • External Interrupts: I/O device ready, timing device, power failure, operator
  • Internal Interrupts (Traps): Register/stack overflow, divide by zero, opcode violation, protection violation
  • Software Interrupts: Supervisor call instruction (user→supervisor mode)

PSW (Processor Status Word) Flags:

C (Carry) = Carry out of MSB from ALU S (Sign) = MSB of ALU output (0=positive, 1=negative) Z (Zero) = 1 if ALU output is all zeros V (Overflow) = 1 if signed overflow occurred = XOR of last two carry bits
TRAP: Overflow flag V is set when result exceeds signed range (e.g., adding two positives gives negative). Sign flag S is simply the MSB of result.

3️⃣ Unit 3 — Input/Output Organization

🔌 I/O Interface COMMANDS

I/O Interface resolves differences between CPU (fast, electronic) and peripherals (slow, electromechanical). 4 types of commands:

  • Control Command: Activates peripheral, tells it what to do
  • Status Command: Tests status conditions of interface/peripheral
  • Data Output Command: Interface transfers data FROM bus INTO its register
  • Data Input Command: Interface receives data FROM peripheral, places in buffer

📟 Isolated vs Memory-Mapped I/O COMPARE MCQ

Isolated I/O (I/O Mapped)

  • Separate address space for I/O
  • Distinct IN/OUT instructions
  • Separate control lines (IOR/IOW)
  • Fewer address lines needed
  • Example: Intel x86

Memory-Mapped I/O

  • I/O and memory SHARE address space
  • Same read/write instructions
  • No separate I/O instructions
  • Flexible, powerful I/O operations
  • Reduces available memory
MCQ TRAP: In memory-mapped I/O, I/O devices and memory share the SAME address space (Answer d in Unit IV MCQ bank Q1). Isolated I/O has separate address spaces.

💾 DMA — Direct Memory Access REGISTERS

DMA Controller takes over buses to manage DIRECT transfer between memory and I/O device, freeing CPU.

DMA Controller Registers: 1. Address Register → starting memory address for transfer 2. Word Count Register → number of words to transfer 3. Control Register → mode: read/write direction CPU initializes DMA then does other work. DMA requests bus → memory controller grants → transfer occurs. Two DMA Transfer Modes: 1. Burst Transfer: Entire block at once (CPU bus denied during transfer) 2. Cycle Stealing: One word at a time (CPU delayed one memory cycle each time)
KEY MCQ: DMA = data transferred WITHOUT CPU intervention. After initialization, CPU is free. Cycle stealing = DMA steals ONE memory cycle at a time from CPU. Bus Grant (BG)=1 means DMA has bus control.

⚡ Priority Interrupts DAISY CHAIN

Software Priority (Polling): CPU polls each device in priority order. Simple but SLOW. Common branch address for all interrupts.

Hardware Priority (Daisy Chain):

Daisy Chain: Devices connected serially. Highest priority = closest to CPU. PI (Priority In) = 1 from CPU → passes down chain PO (Priority Out) = 0 if device HAS pending request (blocks chain) PO (Priority Out) = 1 if device has NO pending request (passes through) Device with PI=1 and NO pending → PO=1 (passes to next) Device with PI=1 and HAS pending → PO=0, PI=0 (stops chain, generates VAD) TRAP: PI=1,PO=1 means device is passing priority through (no interrupt request)
TRAP (from MCQ): In Daisy Chaining, if device has NO pending interrupt: PI=1, PO=1. It receives priority and passes it on. (Answer d: PI=1, PO=1)

📡 Asynchronous Data Transfer STROBE/HANDSHAKE

Strobe method: Single control signal indicates when data is valid. Simple but no acknowledgment.

Handshaking: Two-way exchange. Source sends "data ready", destination sends "data accepted". More reliable.

UART: Universal Asynchronous Receiver Transmitter. Uses start bit (0), data bits, stop bit (1). Baud rate = bits per second.

Asynchronous serial format: Start bit + 8 data bits + 1 stop bit = 10 bits per character. 10 char/sec × 11 bits = 110 baud.

🖥 I/O Processor (IOP/Channel) DEF

IOP = processor with DMA capability that executes its own I/O-specific instructions (channel programs).

  • Accesses memory by cycle stealing
  • Unlike DMA, IOP can FETCH and EXECUTE its own instructions
  • CPU is master; IOP is slave processor
  • CPU initiates channel → IOP operates independently
4️⃣ Unit 4 — RTL, Micro-Operations & Computer Arithmetic

📝 Register Transfer Language (RTL) NOTATION

RTL Basic Symbols: R2 ← R1 (transfer contents of R1 into R2, non-destructive) P: R2 ← R1 (conditional: if P=1, transfer R1 to R2) R2(0-7) (bits 0–7 of register R2) MAR, PC, IR, AC (register names — capital letters) Comma (,) (separates simultaneous micro-operations) Colon (:) (terminates control function) Key: R2 ← R1 means R1 is COPIED to R2. R1 is NOT changed. R2 is REPLACED.

4 Types of Micro-Operations:

  • Register Transfer: R2←R1 (copy)
  • Arithmetic: ADD, SUB, INCREMENT, DECREMENT
  • Logic: AND, OR, XOR, NOT (bit manipulation)
  • Shift: Logical, Circular, Arithmetic shifts

↔️ Shift Micro-Operations 3 TYPES

Logical Shift

  • Vacated bit filled with 0
  • SHL/SHR
  • For unsigned numbers

Circular Shift (Rotate)

  • Shifted-out bit fills vacated end
  • CIL/CIR (through carry E)
  • No bits lost

Arithmetic Shift

  • Preserves sign bit
  • ASHR: fills with sign bit
  • ASHL: fills with 0 (may lose bit)
  • ASHR = divide by 2, ASHL = multiply by 2

Shift Register Types

  • SISO: Serial In, Serial Out
  • SIPO: Serial In, Parallel Out
  • PISO: Parallel In, Serial Out
  • PIPO: Parallel In, Parallel Out

✖️ Booth's Multiplication Algorithm NUMERICAL EXAM

For signed 2's complement multiplication. Strings of 0s or 1s need only shifts (efficient).

Rules (examine current bit Qn and previous bit Q_{n-1}): Qn=0, Q(n-1)=0 → No operation (just shift right) Qn=1, Q(n-1)=1 → No operation (just shift right) Qn=1, Q(n-1)=0 → SUBTRACT multiplicand (AC ← AC - M) Qn=0, Q(n-1)=1 → ADD multiplicand (AC ← AC + M) After each step: Arithmetic Right Shift (AC, Q, Q_{n-1}) Initial: Q_{n-1} = 0, AC = 0 Number of steps = number of bits in multiplier Example: Multiply +7 (0111) × -3 (1101): M = 0111 (multiplicand), Q = 1101 (multiplier) Step 0: Q(n-1)=0, Q0=1 → AC←AC-M, then ASHR Step 1: examine next bit pair and apply rules...
KEY: Start with Q(n-1)=0. Strings of 1s: subtract at first 1 (1,0 transition), add at first 0 after (0,1 transition). More efficient than standard multiplication for signed numbers.

🔢 Number Representations NUMERICAL

Represent +9 and -9 in 7-bit: +9: Signed magnitude = 0 001001 (all three same) -9: Signed magnitude = 1 001001 Signed 1's complement = 1 110110 (flip all non-sign bits) Signed 2's complement = 1 110111 (1's complement + 1) 2's complement subtraction: A - B = A + (2's complement of B) Carry out of MSB in 2's complement is DISCARDED Overflow = XOR of last two carries = 1 means overflow occurred

📡 HDL (Hardware Description Language) DEF

Two main HDLs: VHDL and Verilog. HDL Design Flow: Design → Simulation → Synthesis → Verification.

EDA Playground: Online simulator. Left window = Testbench (inputs/checking). Right window = Design (Verilog circuit).

HDL vs Software: HDL describes HARDWARE that runs in PARALLEL. Software = sequential execution. HDL code runs simultaneously (concurrent statements).

5️⃣ Unit 5 — Memory Organization & Virtual Memory

📐 Memory Hierarchy SPEED ORDER

Goal: highest speed + lowest cost. Registers are fastest, main memory is middle, secondary storage is slowest.

Speed (fast→slow): Registers > Cache > Main Memory (RAM) > Secondary (HDD/SSD) Cost (high→low): Registers > Cache > Main Memory > Secondary Size (small→large): Registers < Cache < Main Memory < Secondary Memory access speed: Register (1ns) > Cache (5ns) > RAM (50ns) > Disk (5ms)

🗂 Associative Memory (CAM) DEF MCQ

Content Addressable Memory: Accessed by CONTENT, not address. All words searched SIMULTANEOUSLY (parallel).

  • Each word has: Address field + Data field
  • Argument register compared with ALL stored words at once
  • Match register indicates which words matched
  • Used in: Cache memory mapping, TLB (Translation Lookaside Buffer)

💾 Cache Memory NUMERICAL EVERY EXAM

Locality of Reference: Temporal (reuse same data) + Spatial (access nearby data). Cache exploits this.

Hit Ratio (h) = number of cache hits / total memory accesses Effective Access Time: Te = h × Tc + (1-h) × (Tc + Tm) = Tc + (1-h) × Tm Example: Tc=0.4ns, Tm=1.2ns, h=0.85 Te = 0.4 + (1-0.85) × 1.2 = 0.4 + 0.18 = 0.58ns TRAP: Formula is Te = Tc + (1-h)×Tm, NOT h×Tc + (1-h)×Tm (Cache access Tc is always done first, then if miss, ALSO do Tm)

🗺 Cache Mapping Techniques 3 TYPES

TechniqueFlexibilityHardware CostConflict MissesAddress Division
Direct MappingLow (1 cache location per block)Low (RAM lookup)HighTag + Index + Offset
Associative MappingHigh (any block → any line)High (CAM, multiple comparators)LowTag + Offset

Direct Mapping Disadvantage: CONFLICT MISSES — two memory blocks that map to the SAME cache line cannot be in cache simultaneously. Associative mapping eliminates this but needs expensive CAM hardware.

✍️ Cache Write Policies MCQ

Write Through

  • On HIT: Write to BOTH cache AND memory
  • On MISS: Write to memory only
  • Memory always up-to-date
  • Slower (memory access on every write)
  • Simple; good for DMA consistency

Write Back (Copy-Back)

  • On HIT: Write to CACHE only
  • On MISS: Bring block to cache, write there
  • Memory may be stale (dirty bit needed)
  • Faster (fewer memory writes)
  • Must write back on eviction

🌐 Virtual Memory PAGING MCQ

Gives programmer illusion of LARGER memory than physically available. Logical address (virtual) → Physical address (real) via page table.

Virtual Address = Page Number | Page Offset Physical Address = Block/Frame Number | Page Offset Page Table: Maps virtual page number → physical frame number TLB (Translation Lookaside Buffer): Fast cache for page table entries (associative memory) Page Fault: Referenced page NOT in physical memory → OS loads from disk Page Fault Service: 1. Find page on backing store (disk) 2. Find free frame (or use replacement algorithm) 3. Load desired page into frame 4. Restart the process

💿 Auxiliary Memory (Secondary Storage) DEF

Non-volatile, high-capacity, low-cost. NOT directly accessible by CPU. Data retained when power off.

Magnetic Disk Organization: Concentric tracks → sectors → cylinders. Disk access time = Seek time + Rotational latency + Transfer time.

6️⃣ Unit 6 — Pipelining, Parallel Processing & Modern Architecture

🚀 Pipelining SPEEDUP MCQ

Decompose sequential process into sub-operations, each in a dedicated stage, all running CONCURRENTLY.

5-Stage RISC Pipeline: IF → ID → EX → MEM → WB IF = Instruction Fetch (read from memory at PC) ID = Instruction Decode (decode + read registers) EX = Execute (ALU operations) MEM = Memory Access (load/store from/to memory) WB = Write Back (write result to register) Speedup = n × k / (k + n - 1) where k = number of pipeline stages n = number of instructions As n → ∞, Speedup → k (ideal = number of stages) Pipeline Hazards: 1. Structural Hazard: Two instructions need same hardware 2. Data Hazard: Instruction needs result of previous (RAW, WAR, WAW) 3. Control Hazard: Branch instruction changes PC unexpectedly

🖥 Multiprocessors TYPES

Symmetric (SMP)

  • Each CPU has same OS copy
  • Peer-to-peer (no master-slave)
  • All CPUs share memory equally
  • Example: Encore Multimax

Asymmetric

  • Each CPU has predefined task
  • Master-slave relationship
  • Master controls all others
  • Cheaper option

Tightly Coupled

  • Synchronized communication
  • Shared memory system
  • Low overhead

Loosely Coupled

  • Message passing
  • Distributed memory
  • High overhead for data exchange

🎮 GPU Architecture MODERN

  • GPU: Designed for parallel processing of graphics/compute. Hundreds of simpler cores vs CPU's few complex cores.
  • Integrated GPU: On same chip as CPU, shares system RAM
  • Discrete GPU: Separate card with own VRAM
  • NVIDIA CUDA: Parallel computing platform for GPU-accelerated computing
  • GPU advantages: Massively parallel, faster than CPU for parallel tasks (ML, rendering)

📱 ARM & SoC Architecture MODERN

  • ARM: RISC-based, 32/64-bit, low power, used in smartphones/tablets. Fewer transistors = smaller, more efficient.
  • SoC (System on Chip): CPU + GPU + RAM + I/O all on one chip (e.g., Apple A17, Snapdragon)
  • Quantum Computing: Uses qubits (superposition of 0 AND 1). Exponentially faster for specific problems (cryptography, optimization).

🌙 One-Night-Before — 20 Must-Know Points

1. Fetch cycle: T0=AR←PC, T1=IR←M[AR]+PC←PC+1, T2=Decode. Always these 3 steps.

2. PC holds NEXT instruction address. IR holds CURRENT instruction. AR = memory address buffer.

3. Register Reference: opcode=111, I=0. I/O Instructions: opcode=111, I=1. Others = Memory Reference.

4. Direct EA = address field. Indirect EA = M[address field]. PC Relative EA = PC + offset. Indexed EA = IX + offset.

5. Stack PUSH: SP←SP-1, M[SP]←DR. POP: DR←M[SP], SP←SP+1. Stack grows DOWNWARD.

6. RISC = hardwired, fixed-length, load-store only, few modes, single cycle.

7. CISC = microprogrammed, variable-length, memory-to-memory, many modes, multi-cycle.

8. Cache Te = Tc + (1-h)×Tm. Example: Tc=0.4, Tm=1.2, h=0.85 → Te=0.58ns.

9. Direct mapping: conflict misses (same cache line). Associative: most flexible (CAM hardware). Set-associative: compromise.

10. Write Through: always writes memory (consistent). Write Back: only writes cache (faster, needs dirty bit).

11. Virtual Memory: virtual page number → physical frame via page table. Page fault = page not in RAM → load from disk.

12. Page replacement: LRU = least recently used (best practical). OPT = optimal (theoretical). FIFO = simple but Belady's anomaly.

13. DMA: direct memory-to-device transfer without CPU. Programmed I/O: CPU polls (busy). Interrupt I/O: CPU notified.

14. Daisy Chain: highest priority = device closest to CPU. PI=1, PO=0 = device IS requesting interrupt.

15. Booth's Algorithm: 10 transition → SUBTRACT M. 01 transition → ADD M. 00/11 → just ASHR.

16. Flynn's: SISD=traditional CPU. SIMD=GPU/vector. MISD=theoretical. MIMD=multiprocessors.

17. Pipeline speedup → k as n→∞ (k = stages). 5-stage RISC: IF→ID→EX→MEM→WB.

18. Selective Complement = XOR. Selective Set = OR. Selective Clear = AND with complement B. Mask = AND.

19. Memory-Mapped I/O: devices share memory address space. Isolated I/O: separate address space, separate IN/OUT instructions.

20. 3-address: short programs, long instructions. 0-address (stack): long programs, short instructions. 1-address uses implied AC.
